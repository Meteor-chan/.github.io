软件准备

Node.js
Git

配置SSH Keys

1.检查本机的SSH keys的设置

$ cd ~/.ssh

如果提示 No such file or diretory 说明你是第一次使用git

2.生成新的SSH keys：

$ ssh-keygen -t rsa -C “example@email.com”

注意：上面的C为大写，之后的邮箱地址填上你自己的邮箱地址。

接着系统会要求你输入密码，这个将在你提交项目的时候会使用到。如果为空，提交项目时将不使用。

Enter passphrase (empty for no passphrase):<输入加密串>
Enter same passphrase again:<再次输入加密串>

注意：在提交密码时，是没有*字样的，所以直接输入就好
最后看到如下图所示，SSH key 设置就成功了

3.添加SSH key到Github账户

在本机上生成SSH key之后，需要将SSH key添加到Github账户中以完成链接。

  • 首先在C盘Administrator(用户)文件夹下，找到.ssh文件夹，如果找不到文件夹，需要设置显示隐藏文件。在该文件夹下找到id_rsa.pub，使用文本编辑器打开。复制此文件所有内容。
  • 登陆Github账户，打开setting界面，找到SSH and GPG keys项，点击 New SSH key
  • 将在id_rsa.pub文件复制的内容粘贴到Key那一栏，点击Add SSH key完成添加

4.测试

输入以下命令进行测试

$ ssh -T git@github.com

如果显示如下，则输入yes就好

The authenticity of host ‘github.com (207.97.227.239)’ can’t be established.
RSA key fingerprint is 16:27:ac:a5:76:28:2d:36:63:1b:56:4d:eb:df:a6:48.
Are you sure you want to continue connecting (yes/no)?

输入之后，将显示

Hi Meteor-chan! You’ve successfully authenticated, but GitHub does not provide shell access.

配置用户信息

在完成了SSH与Github的链接之后，我们需要设置用户信息。

$ git config –global user.name “name

$ git config –global user.email “example@email.com”

注意: 用户名跟邮箱需要填写真实的。

SSH key成功配置

这样本机便成功连接到github了。

使用Github Pages建立博客

Github Pages通常有两种，第一种是基于你用户名建立的UserName.github.io的组织站，另一种是基于项目的Pages，通常我们搭建博客使用的是第一种。具体步骤如下

  1. 登陆Github账户，创建一个repository，Repository name为username.github.io(此处的username为你的用户名)
  2. 进入该项目，点击Settings，找到Launch automatic page generator并点击，一直下一步即可。
  3. 通过http://username.github.io访问静态网站，如[Meteor-chan.github.io][4]

使用Hexo主题

Hexo是一个简单，快速的博客发布工具，支持Markdown格式

安装Hexo

在Git中输入

$ npm install hexo-cli g

$npm install hexo –save

Hexo初始化

在电脑中新建一个名字叫Hexo的文件夹,在该文件夹下右键打开Git Bash

$ hexo init

$ npm install

接着Hexo会自动在该文件夹建立所需要的文件，安装完之后可以通过http://localhost:4000/查看效果，如果不能查看，也没有关系，可在部署到Github后查看。

安装Hexo插件

  1. npm install hexo-generator-index –save
  2. npm install hexo-generator-archive –save
  3. npm install hexo-generator-category –save
  4. npm install hexo-generator-tag –save
  5. npm install hexo-server –save
  6. npm install hexo-deployer-git –save
  7. npm install hexo-deployer-heroku –save
  8. npm install hexo-deployer-rsync –save
  9. npm install hexo-deployer-openshift –save
  10. npm install hexo-renderer-marked@0.2 –save
  11. npm install hexo-renderer-stylus@0.3 –save
  12. npm install hexo-generator-feed@1 –save
  13. npm install hexo-generator-sitemap@1 –save

Hexo主题设置

这里以主题jacman为例。

1.安装主题

$ hexo clean

$ git clone https://github.com/wuchong/jacman.git themes/jacman

2.启用主题

修改Hexo文件夹下的_config.yml中的theme属性，将其修改为jacman

3.更新主题

$ cd theme/jacman

$ git pull

$ hexo g

部署Hexo到Github

在Hexo文件夹中打开_config.yml文件并进行如下修改

deploy:
type: git
repo: git@github.com:Meteor-chan/Meteor-chan.github.io.git
branch: master

常用Hexo命令

hexo new “postname” # 新建文章
hexo new page “pagename” #新建页面
hexo generator # 生成静态页面至public文件夹
hexo server # 生成预览界面
hexo deploy # 将.deploy目录部署到Gtihub

简写

hexo n == hexo new
hexo g == hexo generator
hexo s == hexo server
hexo d == hexo deploy

使用hexo n “postname”命令可以新建一篇文章,然后通过hexo g -d可以将其部署到Github中。
